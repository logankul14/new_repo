# Hosting Static Website on AWS S3 with index.html and error.html

## Step 1: Create S3 Bucket
1. Open AWS Console → S3 → Create bucket.
2. Enter unique bucket name (e.g., my-static-site-demo).
3. Choose region.
4. Uncheck "Block all public access" (for website hosting).
5. Create bucket.

## Step 2: Upload Website Files
1. Upload `index.html` and `error.html` to your bucket.
2. Ensure both files are in the root of the bucket.

Example index.html:
    <!DOCTYPE html>
    <html>
    <head><title>My Static Site</title></head>
    <body>
        <h1>Hello from AWS S3!</h1>
    </body>
    </html>

Example error.html:
    <!DOCTYPE html>
    <html>
    <head><title>404 Error</title></head>
    <body>
        <h1>Oops! Page Not Found</h1>
        <p>The page you are looking for does not exist.</p>
        <a href="index.html">Back to Home</a>
    </body>
    </html>

## Step 3: Enable Static Website Hosting
1. Go to your bucket → Properties → Static website hosting.
2. Select "Enable".
3. Index document: index.html
4. Error document: error.html
5. Save changes.

## Step 4: Update Bucket Policy for Public Access
1. Go to Permissions → Bucket policy.
2. Add policy (replace BUCKET_NAME with your bucket name):

    {
      "Version": "2012-10-17",
      "Statement": [
        {
          "Sid": "PublicReadGetObject",
          "Effect": "Allow",
          "Principal": "*",
          "Action": "s3:GetObject",
          "Resource": "arn:aws:s3:::BUCKET_NAME/*"
        }
      ]
    }

3. Save.

## Step 5: Test Website
1. Go to Properties → Static website hosting.
2. Copy the **Endpoint URL**, e.g.:
   http://my-static-site-demo.s3-website-us-east-1.amazonaws.com
3. Open in browser.

- Accessing valid page → shows index.html
- Accessing invalid page → shows error.html
